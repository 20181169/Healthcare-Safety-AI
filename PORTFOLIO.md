# Healthcare Safety AI Portfolio

이 프로젝트는 심폐음 진단 보조, 현장 안전 모니터링, 소아 X-ray 판독 보조라는 세 가지 AI 경진대회 경험을 하나의 Django 기반 서비스로 통합한 포트폴리오 프로젝트입니다. 공통 목표는 데이터가 이상적인 조건에서 수집되지 않는 실제 환경을 고려해, 데이터 품질 개선부터 모델 추론, 결과 시각화, 웹 서비스 제공까지 이어지는 End-to-End AI 시스템을 구현하는 것이었습니다.

## 해결하고자 한 문제

헬스케어와 산업안전 분야의 AI 서비스는 단순히 모델 정확도만 높인다고 현장에 적용되기 어렵습니다. 심폐음은 청진 환경에 따라 잡음과 주파수 손실이 발생하고, 공사현장 CCTV는 저조도와 원거리 소형 객체 때문에 안전모 탐지가 불안정하며, 소아 X-ray는 의료 데이터 부족과 클래스 불균형으로 인해 일반적인 이미지 분류 방식만으로 충분한 성능을 확보하기 어렵습니다.

따라서 이 프로젝트에서는 각 도메인의 입력 데이터 품질 문제를 먼저 분석하고, Audio Super-Resolution, 저조도 보정, 초해상도 복원, GAN 기반 데이터 증강, Grad-CAM 설명 가능 AI를 조합했습니다. 이를 통해 기존의 단일 모델 추론 방식보다 현장 데이터 변화에 강한 서비스 구조를 설계했습니다.

## 기술스택 선택 이유

- **Django**: 세 개의 AI 서비스를 하나의 인증, DB, 대시보드, 업로드 흐름으로 통합하기 위해 사용했습니다.
- **PyTorch, torchaudio, librosa**: 심폐음 waveform, spectrogram, log-mel feature 분석과 Audio Super-Resolution, CNN 분류 모델 구현에 적합해 사용했습니다.
- **YOLO, OpenCV**: 현장 이미지와 영상에서 사람, 머리, 안전모 객체를 실시간으로 탐지하고 ROI 기반 후처리를 수행하기 위해 사용했습니다.
- **Zero-DCE++, SRGAN, Real-ESRGAN**: 저조도, 노이즈, 저해상도 CCTV 입력을 보정해 탐지 모델의 입력 품질을 높이기 위해 사용했습니다.
- **PGGAN, CNN 기반 Multi-Label Classification**: 소아 X-ray 데이터 부족과 클래스 불균형을 완화하고, 복수 질환 가능성을 동시에 예측하기 위해 사용했습니다.
- **Grad-CAM**: 의료 영상 AI의 판단 근거를 시각화해 사용자가 모델 결과를 해석할 수 있도록 적용했습니다.

## 1. Heart & Lung Sound AI Challenge, 최우수상

### 문제 정의

가정이나 비전문 환경에서 디지털 청진기로 측정한 심폐음은 주변 소음, 측정 위치 차이, 샘플링 품질 저하로 인해 진단 보조 모델의 입력으로 바로 사용하기 어렵습니다. 단순 분류 모델만 적용하면 정상음, crackle, wheeze, both와 같은 클래스 간 경계가 흐려지고, 고주파 성분 손실로 이상음 특징이 약해질 수 있습니다.

### 접근 방법

심폐음 입력을 먼저 정규화하고, 저품질 신호를 고해상도 신호로 복원하는 Audio Super-Resolution 파이프라인을 구성했습니다. 이후 복원된 waveform을 기반으로 log-mel spectrogram을 생성하고 CNN 분류 모델에 입력했습니다. 짧은 녹음이나 길이가 다른 녹음을 안정적으로 처리하기 위해 6초 단위 segmentation과 overlap-add 복원 방식을 적용했습니다.

### 데이터 분석

- waveform, spectrum, spectrogram을 비교해 주파수 손실 구간과 잡음 패턴을 확인했습니다.
- 16 kHz 기준으로 resampling과 peak normalization을 적용해 입력 스케일을 통일했습니다.
- 심폐음 클래스별 데이터 수와 예측 confidence 분포를 확인해 클래스 불균형과 혼동 클래스를 분석했습니다.
- 원본 고해상도 신호가 있는 경우 입력 SNR과 복원 SNR을 비교해 복원 품질을 정량화했습니다.

### 기존 대비 개선 지표

기존 방식은 저품질 입력을 바로 분류 모델에 넣는 구조였습니다. 개선된 방식은 Audio Super-Resolution을 먼저 적용해 신호 품질을 높인 뒤 분류를 수행합니다.

- **SNR 개선**: `snr_in_db`와 `snr_out_db`를 비교해 복원 전후 신호 품질 향상을 측정했습니다.
- **분류 성능 개선**: baseline 대비 accuracy, macro-F1, recall, top-1 confidence 변화를 비교했습니다.
- **서비스 지표**: 업로드부터 복원 파일 생성, waveform/spectrogram/spectrum 시각화까지의 처리 흐름을 웹에서 확인할 수 있도록 구성했습니다.

포트폴리오 표현 예시:

> 저품질 심폐음 입력을 바로 분류하던 baseline 구조에서 Audio Super-Resolution 기반 복원 단계를 추가해 SNR, macro-F1, recall 변화를 비교했습니다. 이를 통해 단순 모델 학습이 아니라 실제 청진 환경에서 발생하는 입력 품질 저하 문제를 먼저 해결하는 진단 보조 파이프라인을 구현했습니다.

## 2. 객체 추적 및 추정 데이터 활용 현장안전 프로젝트, 우수상

### 문제 정의

공사현장 CCTV는 야간, 역광, 먼지, 원거리 촬영, 모션 블러 등으로 인해 안전모 착용 여부를 안정적으로 판단하기 어렵습니다. 특히 작은 머리 영역이나 저조도 환경에서는 YOLO 단독 탐지만으로 미탐과 오탐이 발생할 수 있습니다.

### 접근 방법

YOLO 기반 객체 탐지를 중심으로 사람과 머리 영역을 검출하고, Head ROI 이중 분류를 통해 안전모 착용 여부를 재검증하는 구조를 설계했습니다. 입력 이미지 품질이 낮은 경우에는 Zero-DCE++를 활용한 저조도 보정과 SRGAN, Real-ESRGAN 기반 초해상도 복원을 적용해 탐지 전 입력 품질을 개선했습니다.

### 데이터 분석

- 조도 구간별 탐지 성능을 나누어 저조도 환경에서의 recall 저하를 분석했습니다.
- bounding box 크기별 성능을 확인해 원거리 소형 객체에서 발생하는 미탐 문제를 파악했습니다.
- 안전모 착용, 미착용 클래스의 분포를 확인하고, 미착용자 recall을 주요 지표로 설정했습니다.
- 이미지별 `n_total`, `n_violation`, `mean_conf`, `elapsed_ms`를 기록해 서비스 관점의 결과를 분석했습니다.

### 기존 대비 개선 지표

기존 방식은 원본 CCTV 이미지를 YOLO 모델에 바로 입력하는 구조였습니다. 개선된 방식은 입력 품질 개선과 ROI 재검증을 추가했습니다.

- **탐지 성능 개선**: mAP@0.5, precision, recall, 미착용자 recall을 비교했습니다.
- **오탐 감소**: Head ROI 이중 분류를 통해 안전모 오탐과 미착용 오탐을 줄이는 방향으로 검증했습니다.
- **저조도 대응**: Zero-DCE++ 적용 전후 평균 confidence와 탐지 성공률을 비교했습니다.
- **서비스 성능**: 프레임별 `elapsed_ms`를 기록해 실시간 처리 가능성을 확인했습니다.

포트폴리오 표현 예시:

> YOLO 단독 탐지의 한계를 보완하기 위해 저조도 보정, 초해상도 복원, Head ROI 재분류를 결합했습니다. 조도, 객체 크기, 안전모 착용 여부별 성능을 분리 분석했고, mAP@0.5, 미착용자 recall, 오탐률, 평균 처리시간을 기준으로 기존 탐지 구조 대비 개선 효과를 검증했습니다.

## 3. 2022 소아 X-ray 데이터 과제 서비스 아이디어 개발 챌린지, 우수상

### 문제 정의

소아 흉부 X-ray는 성인 의료 영상에 비해 공개 데이터가 적고, 질환별 라벨 수가 불균형하며, 판독에는 높은 전문성이 요구됩니다. 또한 응급차나 공공보건의료센터처럼 전문의가 상주하지 않는 환경에서는 원격 판독 보조 시스템의 필요성이 큽니다.

### 접근 방법

PGGAN을 활용해 부족한 X-ray 데이터를 증강하고, 증강 데이터를 포함한 학습셋으로 multi-label classification 모델을 구성했습니다. 하나의 X-ray 이미지에서 여러 질환 가능성을 동시에 예측하도록 설계했으며, Grad-CAM을 적용해 모델이 주목한 폐 영역을 시각화했습니다.

### 데이터 분석

- 질환별 라벨 빈도와 정상/비정상 비율을 확인해 클래스 불균형을 분석했습니다.
- 질환 동시 출현 패턴을 확인해 multi-label 문제로 정의했습니다.
- 원본 이미지와 증강 이미지의 명암, 해상도, 분포 차이를 비교했습니다.
- threshold별 precision, recall 변화를 확인해 서비스에서 사용할 판정 기준을 조정했습니다.

### 기존 대비 개선 지표

기존 방식은 제한된 원본 X-ray 데이터만으로 모델을 학습하는 구조였습니다. 개선된 방식은 PGGAN 증강과 설명 가능한 결과 시각화를 추가했습니다.

- **데이터 부족 완화**: 소수 클래스에 대한 recall과 macro-F1 변화를 비교했습니다.
- **진단 보조 성능**: 질환별 AUC, sensitivity, specificity, precision-recall curve를 확인했습니다.
- **설명 가능성 개선**: Grad-CAM 시각화를 통해 모델 예측 근거를 의료진이 확인할 수 있도록 했습니다.
- **서비스화**: 환자 등록, X-ray 업로드, AI 추론, 전문의 확정 흐름을 Django 서비스로 연결했습니다.

포트폴리오 표현 예시:

> 소아 X-ray 데이터 부족과 클래스 불균형 문제를 해결하기 위해 PGGAN 기반 데이터 증강을 적용하고, multi-label classification 모델을 학습했습니다. 질환별 AUC, macro-F1, sensitivity, specificity를 기준으로 증강 전후 성능을 비교했으며, Grad-CAM을 통해 AI 판단 근거를 시각화해 원격 판독 보조 서비스 아이디어로 확장했습니다.

## 통합 서비스 성과

세 프로젝트를 각각 독립된 모델 실험으로 끝내지 않고 하나의 Django 서비스로 통합했습니다. 사용자는 로그인 후 대시보드에서 소아 X-ray, 심폐음, 현장안전 기능으로 이동할 수 있으며, 각 기능은 업로드, 추론, 결과 저장, 시각화를 포함합니다.

- 심폐음: 복원 음원, 예측 클래스, confidence, SNR, waveform, spectrogram, spectrum 제공
- 현장안전: 탐지 결과 이미지, 위반자 수, 평균 confidence, 처리시간 제공
- 소아 X-ray: 질환별 확률, threshold 기반 positive label, Grad-CAM, 전문의 확정 흐름 제공

이 프로젝트의 핵심은 AI 모델을 단순히 학습하는 것이 아니라, 실제 서비스에서 문제가 되는 데이터 품질, 설명 가능성, 사용자 흐름, 처리시간을 함께 고려했다는 점입니다. 이를 통해 경진대회 아이디어를 실제 웹 서비스 형태로 재구성하고, 포트폴리오에서 기술 선택 이유와 정량 검증 관점을 함께 보여줄 수 있도록 구성했습니다.
