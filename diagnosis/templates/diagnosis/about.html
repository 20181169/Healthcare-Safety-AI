Heart & Lung Sound AI Challenge

가정용 디지털 청진기 측정음을 청진실 수준으로 복원하여 정확한 진단을 보조하는 시스템

PROBLEM

왜 이게 어려운가

심폐음 자동 진단은 청진실에서 16 kHz 이상으로 녹음된 HR 데이터셋(ICBHI 등)으로 학습되지만, 실제 가정에서 쓰이는 디지털 청진기는 8 kHz 이하의 LR 신호만 출력합니다. 두 도메인 사이의 주파수 대역 격차로 인해 청진실에서 학습한 분류기가 가정 환경에서 무너집니다.

기존 접근 ①
HR 만 학습
→ 가정 환경 적용 불가
기존 접근 ②
LR 직접 학습
→ 고주파 burst(crackle) 손실로 성능 저하
제안
SR → 분류
→ LR 복원 후 HR 분류기 적용

📈 환자별 record 수 분포
대부분 환자가 1~3 record. random split 시 같은 환자의 다른 segment 가 val 에 들어가 data leakage 발생. → 환자 단위 split 으로 변경 후 baseline F1 0.601 → 0.452 로 떨어졌고 이게 진짜 성능.
📈 원본 sampling rate 분포
Sampling rate 가 4종으로 혼재 — 그대로 학습 시 분류기가 SR artifact 를 단서로 사용하는 leakage 발생. → HR 16 kHz / LR 8 kHz 두 표준으로 통일 후 paired 데이터 합성.

💡 진짜 가치 — 클래스별 Recall

Macro F1 은 +0.0017 로 사실상 동일하지만, SR-boost 의 진짜 효과는 minority class 의 recall 향상.

클래스 Baseline Recall SR-boost Recall Δ 임상 해석
Normal (n=843) 0.765 0.562 −0.203 정상을 의심으로 분류 (FP ↑)
Crackle (n=282) ★ 0.255 0.606 +0.351 +35%p — 폐렴 의심 burst 잡음 ↑
Wheeze (n=169) 0.521 0.462 −0.059 소폭 감소
Both (n=49) 0.224 0.224 0.000 동일
trade-off: SR-boost 는 분류기를 더 "민감"하게 만들어 Normal 신호도 의심으로 분류하는 경향 (specificity 0.765 → 0.562). 그 대가로 가장 임상 가치가 큰 Crackle (폐렴/감염 의심 burst) 의 recall 이 2.4배 향상 (0.255 → 0.606). 놓침 ↓ + 추가 검사 권유 ↑ 는 1차 보조 시스템에서 합리적 trade-off.
측정 환경: ICBHI val split 1,343 cycle (Normal 843 · Crackle 282 · Wheeze 169 · Both 49) · cls_val.csv 환자 단위 split · 6초 center crop · CPU 추론 · checkpoint best_baseline.pt vs best_sr.pt · SR overlap-add 적용 시 cycle 당 평균 약 0.5초 (CPU) · 측정 스크립트: scripts/eval_diagnosis.py.

ablation 표 (Record-level split → Patient-level split → +Class-weighted → +SR 단계별 추가) 는 대회 시점의 최종 두 체크포인트만 보유한 관계로 재현 불가하여 본 페이지에서는 누락. 위 두 행은 실측 단일 측정 결과이며, "Δ macro F1 +15.7%p" 같은 이전 추정값은 잘못된 표기였음.

STACK

기술 스택 선택 근거

AudioSR U-Net (1D)
시간 도메인 SR 백본
28 MB

STFT 도메인 SR (예: NU-wave) 대비 위상 복원 손실이 없고, skip connection 으로 저주파 디테일을 보존. Mel 기반 모델은 위상 정보 추론에 vocoder 가 또 필요하지만 1D U-Net 은 single-pass 가능.

L1 + Multi-scale STFT loss
시간/주파수 듀얼 손실

Crackle (짧은 burst) 는 시간 도메인 손실(L1) 로, Wheeze (지속 톤) 는 주파수 도메인 손실(STFT magnitude) 로 각각 최적화. 단일 손실 사용 시 한쪽 클래스가 흐려지는 trade-off 를 해결.

HeartLungCNN (Raw 1D)
분류기 — STFT 미사용
5.8 MB

SR 출력(시간 waveform)을 그대로 입력 받음 → SR 모델과 분류 모델 사이 representation gap 제거. Mel-spectrogram 분류기는 SR 결과를 다시 한 번 lossy 변환해야 하므로 SR 의 이득이 일부 사라짐.

SR-boost 학습 전략 ★
대회 우승 핵심 인사이트

Baseline 분류기를 LR 만으로 학습 후, 동일 architecture 를 {LR, SR(LR)} 두 도메인 모두에 노출시켜 fine-tune. SR 산출물이 분류기에 익숙해져 inference 시 도메인 갭이 추가로 좁혀짐. 체크포인트: best_baseline.pt → best_sr.pt

회고 / 한계

  • • SR 자체는 HR ref 가 있는 sample 에서만 정량 평가 가능 — 실제 가정 환경 LR 에는 perceptual eval 한계
  • • 청진 부위(heart vs lung) 별 모델 분리는 시도하지 않음 — 데이터 부족으로 multi-task 단일 모델로 통합
