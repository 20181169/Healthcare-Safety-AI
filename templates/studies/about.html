PROBLEM

왜 응급실에 영상의학과가 필요한가

소아 응급실(특히 야간/지방 소규모 병원)에는 영상의학과 전문의가 상주하지 않는 경우가 많습니다. 응급의는 X-ray 를 촬영해도 즉시 확정 판독을 받지 못하고 영상의학과 전송 → 판독 → 회신 사이클을 기다려야 합니다. 이 지연은 평균 수십 분에서 수 시간이며, 폐렴 같은 시급 케이스에서 적정 치료 시작 시점을 놓치는 원인이 됩니다.

현행 ①
응급실 → 영상의학과 송부 → 회신
평균 지연 30 ~ 120 분 (야간/주말 더 김)
현행 ②
응급의 단독 판독
소아 흉부는 성인과 다른 해부학적 특성 → 비전문 판독 위험
제안
AI 1차 분류 + 영상의학과 원격 확정
응급실은 즉시 1차 결과 받고, 영상의학과는 우선순위 큐 받아 확정
서비스 아이디어 챌린지 핵심 차별점: 단순한 분류기 정확도 경쟁이 아니라 응급의·영상의학과 양쪽 워크플로우를 함께 설계한 점. AI 결과는 항상 "보조"이며 최종 진단은 영상의학과 확정으로 닫히는 3-step 흐름.

STACK

기술 스택 선택 근거

DenseNet-121
분류 백본
28 MB

Stanford CheXNet 에서 X-ray 다중 라벨 분류로 SOTA 달성한 architecture. ResNet 대비 feature reuse 가 좋아 의료영상의 미세 패턴 학습에 유리. 121-layer 가 14-label 다중 분류와 밸런스 좋음 (더 깊은 169 는 과적합).

Multi-label sigmoid + BCE
손실 함수

Softmax+Cross-entropy 는 단일 라벨 분류용 — X-ray 는 한 환자에 여러 소견 동반 가능 (예: Pneumonia + Effusion). 라벨별 독립 sigmoid 가 임상적으로 옳음.

Grad-CAM 시각화
설명 가능성

의료진 신뢰 확보의 핵심. "AI 가 폐렴이라고 했어요" 만으로는 채택 안 됨 — "어디를 보고 그렇게 판단했어요" 를 보여줘야 영상의학과 전문의가 동의/반박할 수 있음. 마지막 conv layer 의 gradient 가중 평균.

3-step 워크플로우 ★
대회 우승 서비스 아이디어

① 응급구조사/응급의 X-ray 업로드 → ② AI 즉시 1차 결과 표시 + Grad-CAM → ③ 영상의학과 전문의가 우선순위 큐에서 검토·확정. 응급실은 1차 결과로 즉시 조치 가능, 영상의학과는 자기 페이스로 확정. 의료기기 규제(AI 단독 진단 금지) 와도 정합.

역할 기반 권한 모델
accounts.User 커스텀

4-role: 응급구조사 / 임상의 / 영상의학과 전문의 / 관리자. 영상의학과만 "확정" 권한 보유 — AI 결과를 의료진이 아닌 사람이 확정 처리하는 사고 방지.

ARCHITECTURE

서비스 워크플로우

1
응급실 업로드
응급구조사·응급의가 X-ray 즉시 업로드. 환자 코드만 필요.
권한: paramedic, clinician
2
AI 1차 분류 + Grad-CAM
DenseNet-121 다중 라벨 분류 → 양성 의심 라벨 추출 → Top-1 Grad-CAM 자동 생성
소요: < 1 초
3
영상의학과 원격 확정
전문의가 우선순위 큐에서 검토 → 최종 라벨 + 중증도 + 코멘트 기록 → 응급실 자동 회신
권한: radiologist, admin
의료기기 규제 정합: 모든 화면에 "AI 보조" 라는 면책 표기. 최종 확정은 라이센스 보유 영상의학과 전문의만 가능. 리포트(/studies/<id>/report/) 에는 확정 의료진 이름·면허번호 자동 인쇄.

회고 / 한계

  • • 14-label 중 Kermany 가 학습 신호를 주는 건 Pneumonia 1개뿐 — 나머지 라벨은 0 마스킹으로 fake positive 차단
  • • 소아 흉선 음영 vs 폐렴 구분이 모델 confusion 의 주 원인 — 영상의학과 라벨링 추가 필요
  • • 실 임상 적용에는 의료기기 인증(SaMD), HIPAA 준수, PACS 연동 등 추가 단계 필요 — 본 작품은 PoC
